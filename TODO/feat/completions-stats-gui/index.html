caco · Completions & Stats — GUI mockup overview

Three candidate layouts for extending crates/caco-gui/src/dialogs/wad_stats.rs from a read-only stats viewer into a full completions/stats manager. All three share the same data model (list of wad_completions rows plus the live wad.stats_snapshot) and the same operations (add, edit notes/date, delete, import/export/clear per-entry stats). They differ only in layout and interaction flow.

A · Two-pane

Completions list on the left, per-map stats for the selected entry on the right.

  • Selection and stats always visible together.
  • Natural home for inline edit/add without a modal-over-modal.
  • Wider dialog; eats horizontal space.

B · Dropdown + action row

Keeps today's dropdown selector; adds contextual Add/Edit/Delete and a meta panel.

  • Smallest diff from today's dialog. Additive only.
  • Familiar shape for existing users.
  • Comparing across entries means re-opening the dropdown.

C · Tabs

Completions tab lists entries with per-row actions; Stats tab shows the selected entry's map table.

  • Every row shows its own Edit / Delete / Import / Export.
  • Cleanest management surface.
  • Viewing stats requires a tab switch — more clicks.

Common behaviors across all three:

  • Add beaten inserts a new wad_completions row (via db::sessions::add_wad_completion) with the current timestamp; opens edit mode so the user can set notes/date.
  • Edit allows changing date + notes on an existing completion row (via a new update_wad_completion_meta helper or extension of update_wad_completion).
  • Delete calls db::sessions::delete_wad_completion after confirmation.
  • Import attaches a parsed stats file to the selected completion (today it only attaches to the latest).
  • Export writes the selected entry's snapshot to disk, matching its format.
  • Clear nulls the selected entry's stats_snapshot.
  • Changes trigger the existing WadStatsResult::Modified so the caller can refresh.
